NEW YORK TAXI FARE PREDICTION
¶

INTRODUCTION¶

The goal in this Kaggle competition was to predict the taxi fares paid by customers in New York using a large data set of historical transactions. The raw data came with the following features:

  • key
  • fare_amount
  • pickup_datetime
  • pickup_longitude
  • pickup_latitude
  • dropoff_longitude
  • dropoff_latitude
  • passenger_count

We will initally build a simple linear model with a distance feature calculated from the absolute distance between the longitude and latitudes of the pickup and drop off points respectivley. This will serve as a benchmark to see if we can use additional feature engineering and a more advanced machine learning algorithm to improve on the results.

CREATE INITIAL NEW FEATURES & REMOVE OUTLIERS¶

Initially after thinking through the problem and seeing the included features it is obvious that there are additional features we can create from the included features that could be predictive of the fare a customer paid:

  • The distance and direction of travel between a pickup and dropoff location
  • The time of day, day of week and month
  • The area of the city a customer was picked up from or dropped off rather than using a single point location

CREATE DISTANCE FEATURE

To determine the distance between the pickup and drop off points we can create a new feature by writing a function that uses the pickup and dropoff coordinates to calculate the geodesic distance.

In [410]:
from math import sin, cos, sqrt, atan2, radians, degrees
def get_distance(x1,x2,x3,x4):
    R = 6373.0
    lat1 = radians(x1)
    lon1 = radians(x2)
    lat2 = radians(x3)
    lon2 = radians(x4)
    dlon = lon2 - lon1
    dlat = lat2 - lat1
    a = sin(dlat / 2)**2 + cos(lat1) * cos(lat2) * sin(dlon / 2)**2
    c = 2 * atan2(sqrt(a), sqrt(1 - a))
    distance = R * c
    return distance

train['distance'] = train.apply(lambda x: get_distance(x['pickup_latitude'],x['pickup_longitude'],
     x['dropoff_latitude'],x['dropoff_longitude']), axis=1)
test['distance'] = test.apply(lambda x: get_distance(x['pickup_latitude'],x['pickup_longitude'],
     x['dropoff_latitude'],x['dropoff_longitude']), axis=1)

CREATE BEARING FEATURE

Using a similar methodology to creating the distance feature, we create a bearing feature that describes the direction of travel and then assign this to one of eight groupings that are 45 degrees apart.

In [411]:
def get_bearing(x1,x2,x3,x4):
    lat1 = radians(x1)
    lon1 = radians(x2)
    lat2 = radians(x3)
    lon2 = radians(x4)
    dlon = lon2 - lon1
    x = sin(dlon) * cos(lat2)
    y = cos(lat1) * sin(lat2) - sin(lat1) * cos(lat2) * cos(dlon)
    bearing = (degrees(atan2(x, y)) + 360) % 360
    return bearing

train['bearing'] = train.apply(lambda x: get_bearing(x['pickup_latitude'],x['pickup_longitude'],
     x['dropoff_latitude'],x['dropoff_longitude']), axis=1)
test['bearing'] = test.apply(lambda x: get_bearing(x['pickup_latitude'],x['pickup_longitude'],
     x['dropoff_latitude'],x['dropoff_longitude']), axis=1)

CLUSTERING¶

The pickup and dropoff locations are given as coordinates and therefore are continous values which are unsuitable for modelling purposes. To get around this we will cluster these locations into 50 groups that should represent different areas around New York.

To do this we will use the K-means clustering algorithm to generate the clusters on a subset of the train dataset and then use a K-Nearest Neighbours algorithm to assign clusters to the test set. The K-Nearest Neighbours alorigthm is usually fast to train and works well when dealing with location data.

FUNCTION TO ASSIGN CLUSTER BASED ON LOCATION COORDINATES¶

def assign_clusters(data, coordinates,cluster_model):
    clusters = cluster_model.predict(coordinates)   
    clusters = pd.Series(clusters)  
    data.reset_index(inplace=True, drop=True)
    data = pd.concat([data, clusters], axis=1)
    return data

TRAIN XGBOOST MODEL USING GRID SEARCH TO FIND HYPERPARAMETERS¶

For this project we will use XGBoost to predict the target variable. XGBoost is an gradient boosting method that has proven to be accurate over a number of different types of prediction competitions and is optimised for multithreading meaning training and prediction is much faster than comparable models.

Boosting models train using a random selection of fields and training examples to build trees that are optimised to reduce the error between the output of the tree for the training examples and training target. The model builds trees sequentially and as each tree is built the examples with the largest errors are given larger weights which ensures the next tree built effectivly pays more attention to these examples when optimising. Any prediction example is effectively passed through all the trees in the final model with the output of each tree either averaged (regression) or voted (classification) to produce a prediction.

Conclusion¶

We have seen that by using XGBoost and additional feature engineering we have been able to improve the RMSE from \$9.76 to \\$4.76 against the benchmark linear model. This is using only a 1% sample of the provided training data to build the model and test.

As a following stage to improve the accuracy of this model we could look to include some or all of the following:

  • Analyse test examples with the largest errors to determine any visible patterns and if these can be used to create new features
  • Use a larger sample of the provided training data to train the model
  • Train a deep learning model which have been shown to provide significant improvements in many cases
